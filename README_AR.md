# تطبيق الأستاذ محمود الدياب للرياضيات - Flutter + Firebase

هذه نسخة بداية احترافية للتطبيق بالخيار الثاني: **Flutter + Firebase**.

## ماذا تحتوي النسخة؟

- تسجيل دخول بالبريد وكلمة المرور عبر Firebase Authentication.
- لا توجد كلمات مرور داخل الكود أو الملفات.
- صلاحيات:
  - Teacher / Admin: يرى كل البيانات ويضيف الطالبات والإعلانات.
  - Student: ترى بياناتها وشعبتها فقط.
  - Pending: حساب ينتظر تفعيل الأستاذ.
- أقسام جاهزة:
  - الطالبات
  - الدروس
  - الاختبارات
  - النتائج
  - الحضور
  - الإعلانات

## طريقة التشغيل المختصرة

1. أنشئ مشروع Firebase جديد.
2. فعّل Authentication > Email/Password.
3. فعّل Firestore Database.
4. ثبّت FlutterFire CLI إن لم يكن مثبتًا.
5. داخل مجلد المشروع شغّل:

```bash
flutter pub get
flutterfire configure
flutter run
```

6. انسخ محتوى ملف `firestore.rules` إلى Firebase Console > Firestore Database > Rules ثم انشر القواعد.

## إنشاء حساب الأستاذ

1. من Firebase Console > Authentication أضف مستخدمًا ببريدك الحقيقي وكلمة مرور تختارها أنت.
2. من Firestore أنشئ مستندًا داخل collection باسم `users`.
3. اجعل اسم المستند هو بريدك تمامًا بالأحرف الصغيرة، مثال:

```text
mahmoud@example.com
```

4. أضف الحقول التالية:

```json
{
  "email": "mahmoud@example.com",
  "fullName": "الأستاذ محمود الدياب",
  "role": "Teacher",
  "active": true,
  "classId": "ALL"
}
```

## طريقة الطالبات

- الطالبة تنشئ حسابها من شاشة التطبيق.
- إذا لم تكن مضافة من الأستاذ تظهر لها رسالة: "الحساب بانتظار التفعيل".
- الأستاذ يضيف بريد الطالبة من صفحة الطالبات داخل التطبيق، ثم تصبح قادرة على رؤية بياناتها.

## ملاحظات مهمة

- استخدم البريد بالأحرف الصغيرة دائمًا.
- لا تضع كلمات مرور الطالبات في أي ملف.
- هذا مشروع بداية MVP، وبعد تشغيله يمكن إضافة:
  - رفع ملفات PDF.
  - بنك أسئلة تفاعلي.
  - إشعارات Push Notifications.
  - لوحة إدارة متقدمة.
  - نشر على Google Play.
