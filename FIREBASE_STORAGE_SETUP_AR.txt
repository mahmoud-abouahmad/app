تحديث رفع ملفات الدروس من الهاتف — v1.4

الميزات:
- اختيار ملف من الهاتف داخل نافذة إضافة أو تعديل الدرس.
- دعم PDF والصور وWord وPowerPoint.
- الحد الأقصى 20 ميغابايت.
- شريط تقدم أثناء الرفع.
- حفظ رابط الملف واسم الملف تلقائيًا.
- استبدال أو إزالة الملف الحالي.
- حذف الملف من Firebase Storage عند حذف الدرس.
- الإبقاء على خيار رابط خارجي ورابط فيديو.
- الضغط على بطاقة الدرس يفتح المحتوى عندما يوجد ملف واحد أو فيديو واحد.
- رقم الإصدار: 1.2.0+8.

الملفات المطلوب استبدالها:
- lib/main.dart
- pubspec.yaml

ملف جديد:
- storage.rules

إعداد Firebase Storage — مرة واحدة فقط:
1. افتح Firebase Console للمشروع.
2. من Build اختر Storage.
3. اضغط Get started وأنشئ مساحة التخزين.
4. افتح تبويب Rules.
5. احذف القواعد الموجودة والصق محتوى storage.rules.
6. اضغط Publish.

مهم:
قد تطلب Firebase ترقية المشروع إلى خطة Blaze لتفعيل Cloud Storage.
لا يكفي رفع storage.rules إلى GitHub؛ يجب نشر محتواه من Firebase Console.

الرفع عبر GitHub Desktop:
1. انسخ محتويات مجلد mahmoud_fix_v1_4 إلى جذر مشروع app.
2. اختر استبدال الملفات.
3. Commit:
   Add direct lesson file upload with Firebase Storage
4. Push origin.
5. بعد نجاح Actions حمّل APK وثبّت نسخة arm64 على Redmi Note 13.

الاختبار:
- أضف درسًا.
- اضغط اختيار ملف من الهاتف.
- اختر PDF صغيرًا.
- تأكد من وصول شريط التقدم إلى النهاية.
- ادخل بحساب طالبة من المجموعة نفسها.
- افتح الدرس واضغط فتح الملف.
